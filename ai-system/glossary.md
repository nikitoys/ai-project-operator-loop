# Glossary — AI Development System

Status: Draft

This glossary defines the terms used by the AI Development System.

The glossary is split into sections:

- `glossary-core.md` — core system terms.
- `glossary-project.md` — product and project documentation terms.
- `glossary-execution.md` — task, review and execution terms.
- `glossary-evolution.md` — system evolution and change-management terms.

## Lifecycle Terms

Lifecycle terms are defined in the glossary section closest to their use:

- `glossary-execution.md` defines task, Codex execution, review and QA lifecycle terms.
- `glossary-evolution.md` defines governance, change, decision, improvement, knowledge and experiment lifecycle terms.

The lifecycle documents are the source of truth for detailed lifecycle rules:

- `lifecycle-governance.md` — shared lifecycle governance model.
- `task-lifecycle.md` — managed task lifecycle.
- `codex-lifecycle.md` — Codex execution lifecycle.
- `review-lifecycle.md` — review and re-review lifecycle.
- `qa-lifecycle.md` — QA and regression lifecycle.
- `change-lifecycle.md` — managed system change lifecycle.
- `decision-lifecycle.md` — Human Owner decision lifecycle.
- `improvement-lifecycle.md` — improvement triage and conversion lifecycle.
- `knowledge-lifecycle.md` — managed knowledge and lessons learned lifecycle.
- `experiment-lifecycle.md` — managed experiment lifecycle.

If a glossary definition and a lifecycle document conflict, the lifecycle document controls the detailed rule and the glossary should be updated.

## Core Principle

All roles must use the same vocabulary.

If a term is unclear, it should be added to the glossary before it becomes part of the workflow or documentation.
